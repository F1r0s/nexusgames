How the system works:
1. GitHub Actions runs auto-fetcher.js monthly → scrapes an1.com → pushes new rows to Google Sheets.
2. generate-games-json.js pulls the Google Sheet CSV → writes games.json → commits it to GitHub.
3. server.js loads games.json on startup (instant) → serves /api/games.
4. The frontend (index.html) fetches /api/games and renders the game grid.
5. Edits you make here write directly to games.json (live, no restart needed).

Display order: Legendary → New → Hot → General. Drag rows in the Games table to reorder manually.

Data Sync

Google Sheets Sync

Pulls the latest data from your public Google Sheet CSV and overwrites games.json in memory. The server will serve the new data immediately.

Automation pipeline:
• GitHub Actions runs on the 1st of every month (or manually via Actions tab).
• It runs node auto-fetcher.js → scrapes an1.com → pushes new rows to your Google Sheet.
• Then runs node scripts/update-month.js → updates month strings in HTML files.
• Finally commits games.json, index.html, and best-games.html back to the repo.

Google Sheet columns:
Module Name | Version | Build Data | Size | OS | Architecture | Tags | Visual Asset | Access Link | Data Log
